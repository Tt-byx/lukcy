## 步骤 1：后端 — 新增"获取历史活动" API

### 1.1 ActivityService.java

新增方法签名：

```java
List<Activity> getHistoryActivities();
```

### 1.2 ActivityServiceImpl.java

新增实现（只查 status = 2 已结束的活动，按创建时间倒序）：

```java
@Override
public List<Activity> getHistoryActivities() {
    return list(new LambdaQueryWrapper<Activity>()
            .eq(Activity::getStatus, 2)
            .orderByDesc(Activity::getCreatedAt));
}
```

### 1.3 ActivityController.java

新增接口 `GET /api/activity/history`：

```java
@GetMapping("/history")
public Result<List<Activity>> getHistory() {
    return Result.ok(activityService.getHistoryActivities());
}
```

> 说明：已有 `GET /api/lottery/rounds?activityId=` 和 `GET /api/lottery/winners?roundId=` 两个接口，历史活动页直接复用这两个。

### 3.4 新增函数

```javascript
// 获取历史活动列表
async function fetchHistoryActivities() {
  try {
    const res = await fetch(`${API_BASE}/api/activity/history`)
    const data = await res.json()
    if (data.code === 200) {
      historyActivities.value = data.data
    }
  } catch (e) {
    console.error('获取历史活动失败:', e)
  }
}

// 展开/收起历史活动详情
async function toggleHistory(activity) {
  if (historySelectedId.value === activity.id) {
    historySelectedId.value = null
    return
  }
  historySelectedId.value = activity.id
  // 加载该活动的轮次
  try {
    const res = await fetch(`${API_BASE}/api/lottery/rounds?activityId=${activity.id}`)
    const data = await res.json()
    if (data.code === 200) {
      historyRounds.value = data.data
      // 依次加载每个轮次的中奖者
      for (const round of data.data) {
        const winRes = await fetch(`${API_BASE}/api/lottery/winners?roundId=${round.id}`)
        const winData = await winRes.json()
        if (winData.code === 200) {
          historyWinners.value[round.id] = winData.data
        }
      }
    }
  } catch (e) {
    console.error('加载历史活动详情失败:', e)
  }
}
```

### 3.6 中奖记录名称展示优化

后端 `Winner` 实体只有 `participantId`，没有参与者姓名。需要在加载中奖记录时，同时查询参与者姓名。但为了简化，先用 `participantId` 展示，后续可优化。

> 更好的做法：后端新增一个 DTO 或 VO 来 join 查询优胜者和参与者姓名，或在前端用 `GET /api/participant/list?activityId=` 一次性拿到所有参与者做映射。这里先保持简洁，后续有需要再优化。

## 边界情况处理

* 没有历史活动 → 显示"暂无历史活动"

* 历史活动没有轮次 → 显示"该活动没有抽奖记录"

* 历史活动轮次无中奖者 → 不显示中奖者区域

* Winner 实体只有 participantId 无姓名 → 先展示 ID，后续可优化 join 查询
