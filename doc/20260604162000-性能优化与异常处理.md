### 1. 创建全局异常处理器

#### 1.1 创建 BusinessException 自定义业务异常类
**文件：** `lucky-server/src/main/java/com/lucky/exception/BusinessException.java`

```java
@Getter
public class BusinessException extends RuntimeException {
    private final int code;

    public BusinessException(String message) {
        super(message);
        this.code = 500;
    }

    public BusinessException(int code, String message) {
        super(message);
        this.code = code;
    }

    public BusinessException(String message, Throwable cause) {
        super(message, cause);
        this.code = 500;
    }
}
```

#### 1.2 创建 GlobalExceptionHandler 全局异常处理器
**文件：** `lucky-server/src/main/java/com/lucky/exception/GlobalExceptionHandler.java`

处理的异常类型：
- `BusinessException` - 业务异常，返回对应的错误码和消息
- `MethodArgumentNotValidException` - 参数校验异常，返回400
- `BindException` - 参数绑定异常，返回400
- `RuntimeException` - 运行时异常，返回通用错误信息
- `Exception` - 其他异常，返回系统错误

**核心代码：**
```java
@Slf4j
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(BusinessException.class)
    public Result<Void> handleBusinessException(BusinessException e) {
        log.warn("业务异常: {}", e.getMessage());
        return Result.error(e.getCode(), e.getMessage());
    }

    @ExceptionHandler(RuntimeException.class)
    public Result<Void> handleRuntimeException(RuntimeException e) {
        log.error("运行时异常", e);
        return Result.error("服务器内部错误，请稍后重试");
    }
}
```

### 4. 添加敏感词缓存

#### 4.1 修改 SensitiveWordServiceImpl
**文件：** `lucky-server/src/main/java/com/lucky/service/impl/SensitiveWordServiceImpl.java`

**核心改动：**
```java
/**
 * 敏感词缓存（线程安全）
 */
private volatile List<String> cachedWords = null;

/**
 * 获取敏感词列表（带缓存）
 */
@Override
public List<String> getAllWords() {
    if (cachedWords == null) {
        synchronized (this) {
            if (cachedWords == null) {
                loadCache();
            }
        }
    }
    return cachedWords;
}

/**
 * 加载缓存
 */
private void loadCache() {
    List<String> words = list().stream()
            .map(SensitiveWord::getWord)
            .map(String::toLowerCase)
            .collect(Collectors.toCollection(CopyOnWriteArrayList::new));
    cachedWords = words;
    log.info("敏感词缓存已加载，共 {} 个词", words.size());
}

/**
 * 清除缓存（增删改时调用）
 */
public void clearCache() {
    cachedWords = null;
}
```

**重写增删改方法，自动清除缓存：**
- `save()` - 添加敏感词时清除缓存
- `removeById()` - 删除敏感词时清除缓存
- `removeByIds()` - 批量删除敏感词时清除缓存

## 为什么这样做

### 1. **全局异常处理**
- **问题：** 未捕获的异常导致500页面暴露堆栈信息
- **解决：** 统一异常处理，返回友好的错误信息
- **收益：** 
  - 用户看到友好的错误提示
  - 开发者通过日志排查问题
  - 避免敏感信息泄露

### 2. **抽奖并发控制**
- **问题：** 两个管理员同时抽奖可能重复中奖
- **解决：** 使用乐观锁，version 字段冲突时提示重试
- **收益：** 
  - 保证抽奖公平性
  - 避免数据不一致
  - 用户体验友好（提示"请稍后重试"）

### 3. **敏感词缓存**
- **问题：** 每次发弹幕都查库，2000人高频发送时性能差
- **解决：** 启动时加载到内存，增删改时自动更新
- **收益：** 
  - 减少数据库查询
  - 弹幕发送更流畅
  - 内存占用可控（敏感词数量有限）

### 4. **简化 Controller**
- **问题：** 每个 Controller 都有 try-catch，代码重复
- **解决：** 统一由全局异常处理器处理
- **收益：** 
  - 代码更简洁
  - 异常处理更统一
  - 维护成本降低

## 注意事项

### 1. **乐观锁使用**
- 需要数据库表有 version 字段
- 更新时必须传入正确的 version 值
- 更新失败时抛出异常

### 2. **敏感词缓存**
- 启动时自动加载
- 增删改时自动清除
- 使用 volatile + 双重检查锁保证线程安全

### 3. **异常处理**
- BusinessException 用于业务异常
- 其他异常由全局处理器统一处理
- 日志记录详细错误信息
