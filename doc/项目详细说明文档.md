# 项目详细说明文档

本文档包含大学毕业典礼抽奖程序的完整技术说明和部署指南。

---

## 1. 项目概述

### 1.1 项目名称
大学毕业典礼抽奖程序

### 1.2 项目简介
这是一款专为大学毕业典礼设计的互动抽奖程序，支持2000人规模的在线实时互动。系统采用三端架构，通过大屏幕展示活动信息和抽奖过程，手机扫码参与互动，后端管理系统统一管理活动流程。该程序旨在增强毕业典礼的互动性和趣味性，让毕业生通过扫码、发弹幕、参与抽奖的方式共同庆祝这一重要时刻。

### 1.3 系统架构

#### 三端架构
```
┌──────────────────────────────────────────────────────────────┐
│                      用户终端层                               │
│                                                              │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐        │
│  │ lucky-screen │  │ lucky-mobile │  │ lucky-admin  │        │
│  │   大屏幕端    │  │    手机端     │  │   后端管理    │        │
│  │  (展示层)     │  │  (参与层)     │  │  (管理层)    │        │
│  └──────────────┘  └──────────────┘  └──────────────┘        │
│         │                 │                 │                │
│         └─────────────────┼─────────────────┘                │
│                           │ HTTP / WebSocket                 │
│                           ▼                                  │
│              ┌────────────────────────┐                      │
│              │   Spring Boot 3.2.5    │                      │
│              │   后端服务层           │                      │
│              └────────────────────────┘                      │
└──────────────────────────────────────────────────────────────┘
```

#### 1. **大屏幕端（lucky-screen）** - 展示层
- **技术栈**：Vue 3 + Vite + WebSocket
- **功能定位**：面向全体师生展示活动信息和抽奖过程
- **核心功能**：
  - 动态二维码展示：实时生成和显示手机端扫描入口
  - 弹幕实时展示：全屏弹幕效果，支持平滑滚动动画
  - 抽奖动画：滚动名字效果、中奖展示
  - 实时统计：参与人数、在线人数统计
  - 动态背景：支持图片/视频背景自定义

#### 2. **手机端（lucky-mobile）** - 参与层
- **技术栈**：Vue 3 + Vite + WebSocket
- **功能定位**：面向参与者（毕业生），扫码进入直播间
- **核心功能**：
  - 快速注册：输入学号快速进入活动
  - 实时弹幕发送：支持文本弹幕和快捷消息
  - 直播间体验：实时消息流、在线互动
  - 参与抽奖：系统自动抽取参与者

#### 3. **后端管理（lucky-admin）** - 管理层
- **技术栈**：Vue 3 + Vite
- **功能定位**：面向活动管理者，控制活动全流程
- **核心功能**：
  - 活动管理：创建/编辑/启动/停止活动
  - 参与者管理：查看参与者列表、管理禁言/移除
  - 抽奖管理：创建多轮抽奖、控制抽奖流程、公布中奖结果
  - 弹幕管理：审核弹幕内容、敏感词管理
  - 屏幕配置：自定义大屏背景、弹幕样式、手机端背景
  - 历史活动：查看历史活动和中奖记录

## 4. 数据库设计

### 4.1 核心表结构

#### activity（活动表）
```sql
CREATE TABLE activity (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL COMMENT '活动名称',
    status TINYINT DEFAULT 0 COMMENT '0未开始 1进行中 2已结束',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### participant（参与者表）
```sql
CREATE TABLE participant (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    activity_id BIGINT NOT NULL,
    name VARCHAR(50) NOT NULL COMMENT '姓名',
    phone VARCHAR(20) COMMENT '手机号',
    student_id VARCHAR(30) COMMENT '学号',
    status TINYINT DEFAULT 1 COMMENT '1已参与 0已中奖 3已移除抽奖',
    is_muted TINYINT DEFAULT 0 COMMENT '是否禁言 0否 1是',
    is_banned TINYINT DEFAULT 0 COMMENT '是否被移除直播间 0否 1是',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### lottery_round（抽奖轮次表）
```sql
CREATE TABLE lottery_round (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    activity_id BIGINT NOT NULL,
    round_name VARCHAR(100) COMMENT '轮次名称，如：一等奖、二等奖',
    winner_count INT DEFAULT 1 COMMENT '本轮抽取人数',
    status TINYINT DEFAULT 0 COMMENT '0未开始 1已完成',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### winner（中奖记录表）
```sql
CREATE TABLE winner (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    round_id BIGINT NOT NULL,
    participant_id BIGINT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### danmaku（弹幕表）
```sql
CREATE TABLE danmaku (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    activity_id BIGINT NOT NULL,
    participant_id BIGINT COMMENT '发送者',
    content VARCHAR(200) NOT NULL COMMENT '弹幕内容',
    status TINYINT DEFAULT 0 COMMENT '0待审核 1已通过 2已拒绝',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### sensitive_word（敏感词表）
```sql
CREATE TABLE sensitive_word (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    word VARCHAR(50) NOT NULL COMMENT '敏感词',
    UNIQUE INDEX uk_word (word)
);
```

#### screen_config（屏幕配置表）
```sql
CREATE TABLE screen_config (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    background_type VARCHAR(20) COMMENT '背景类型: image/video',
    background_url VARCHAR(500) COMMENT '背景文件URL',
    danmaku_area VARCHAR(20) DEFAULT 'full' COMMENT '弹幕显示区域: full/top/bottom',
    danmaku_opacity INT DEFAULT 80 COMMENT '弹幕不透明度(0-100)',
    danmaku_font_size INT DEFAULT 28 COMMENT '弹幕字号',
    danmaku_speed INT DEFAULT 10 COMMENT '弹幕滚动速度(秒)',
    mobile_background_type VARCHAR(20) COMMENT '手机端背景类型',
    mobile_background_url VARCHAR(500) COMMENT '手机端背景URL',
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

## 6. Docker Compose 部署方案

### 6.1 docker-compose.yml
```yaml
version: '3.8'

services:
  # MySQL 数据库
  mysql:
    image: mysql:8.0
    container_name: lucky-mysql
    environment:
      MYSQL_ROOT_PASSWORD: ${DB_PASSWORD}
      MYSQL_DATABASE: lucky
    ports:
      - "3306:3306"
    volumes:
      - mysql-data:/var/lib/mysql
      - ./lucky-server/src/main/resources/schema.sql:/docker-entrypoint-initdb.d/init.sql
    networks:
      - lucky-network

  # 后端服务
  lucky-server:
    build: ./lucky-server
    container_name: lucky-server
    environment:
      - SPRING_PROFILES_ACTIVE=prod
      - DB_HOST=mysql
      - DB_PASSWORD=${DB_PASSWORD}
    ports:
      - "8080:8080"
    depends_on:
      - mysql
    networks:
      - lucky-network

  # Nginx（前端服务）
  nginx:
    image: nginx:alpine
    container_name: lucky-nginx
    ports:
      - "80:80"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf
      - ./lucky-screen/dist:/usr/share/nginx/html/screen
      - ./lucky-mobile/dist:/usr/share/nginx/html/mobile
      - ./lucky-admin/dist:/usr/share/nginx/html/admin
    depends_on:
      - lucky-server
    networks:
      - lucky-network

volumes:
  mysql-data:

networks:
  lucky-network:
    driver: bridge
```

### 6.2 Nginx 配置（nginx/nginx.conf）
```nginx
events {
    worker_connections 1024;
}

http {
    include       /etc/nginx/mime.types;
    default_type  application/octet-stream;

    server {
        listen 80;

        # 大屏幕端
        location /screen {
            alias /usr/share/nginx/html/screen;
            try_files $uri $uri/ /screen/index.html;
        }

        # 手机端
        location /mobile {
            alias /usr/share/nginx/html/mobile;
            try_files $uri $uri/ /mobile/index.html;
        }

        # 管理后台
        location /admin {
            alias /usr/share/nginx/html/admin;
            try_files $uri $uri/ /admin/index.html;
        }

        # API 反向代理
        location /api/ {
            proxy_pass http://lucky-server:8080/api/;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
        }

        # WebSocket 反向代理
        location /ws/ {
            proxy_pass http://lucky-server:8080/ws/;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";
            proxy_set_header Host $host;
        }
    }
}
```


```bash

echo "=== 开始部署项目 ==="

./build.sh

# 停止旧容器
```

## 8. 部署步骤

```bash
cd lucky-screen && npm run dev
cd lucky-mobile && npm run dev
cd lucky-admin && npm run dev
```

### 8.2 服务器生产环境
```bash
# 1. 上传项目到服务器
scp -r ./lucky root@服务器IP:/opt/lucky

# 2. 进入项目目录
cd /opt/lucky

# 3. 设置环境变量
export DB_PASSWORD=your_password

# 4. 一键部署
chmod +x deploy.sh
./deploy.sh

# 5. 查看日志
docker-compose logs -f
```

## 12. 通信流程

### 12.1 实时通信
1. **手机端 → 后端**：HTTP REST API（注册、发送弹幕）
