# 20260520010500-Admin端手机背景图片裁剪功能

## 问题提示词

用户反馈：
1. 手机端背景图片裁剪有很大问题
2. 手机尺寸差不多6.67英寸，需要做适配
3. 如果图片实在不行，建议做裁剪功能
4. 背景上传只能在admin端改变，用户不能自己改

## 做了什么事情

### 1. 引入 cropper.js 裁剪库

**修改文件**：`lucky-admin/index.html`

**修改内容**：
- 在 `<head>` 中添加 cropperjs 的 CSS 和 JS CDN 链接

```html
<link href="https://cdn.jsdelivr.net/npm/cropperjs@1.6.1/dist/cropper.min.css" rel="stylesheet">
<script src="https://cdn.jsdelivr.net/npm/cropperjs@1.6.1/dist/cropper.min.js"></script>
```

### 2. 添加图片裁剪功能

**修改文件**：`lucky-admin/src/App.vue`

**添加的状态变量**：
```javascript
const showMobileCropper = ref(false)      // 是否显示裁剪器
const mobileCropImageUrl = ref('')        // 裁剪预览URL
const mobileCropper = ref(null)           // Cropper实例
const mobileCropperImage = ref(null)      // 裁剪图片元素引用
```

**修改的逻辑**：

1. **文件选择逻辑**（`onMobileFileSelected`）：
   - 当选择图片类型时，使用 FileReader 读取图片并显示裁剪器
   - 当选择视频类型时，直接设置文件（不裁剪）

2. **新增函数**：
   - `initMobileCropper()`：初始化裁剪器，设置比例为 9:20（手机屏幕比例）
   - `confirmCrop()`：确认裁剪，获取裁剪后的 Canvas，转换为 Blob，自动上传
   - `cancelCrop()`：取消裁剪，关闭模态框，清除文件

3. **裁剪配置**：
   - `aspectRatio: 9 / 20`：固定手机屏幕比例
   - `viewMode: 1`：限制裁剪框不超过图片范围
   - `dragMode: 'move'`：可以拖动图片调整位置
   - `autoCropArea: 1`：初始裁剪框占满整个图片
   - 输出尺寸：1080 x 2400 像素（手机分辨率）

**添加的模板**：
```html
<!-- 图片裁剪模态框 -->
<div v-if="showMobileCropper" class="cropper-modal">
  <div class="cropper-modal-content">
    <h4>裁剪手机端背景（比例 9:20）</h4>
    <p class="cropper-hint">拖动裁剪框选择显示区域，手机端将显示裁剪后的图片</p>
    <div class="cropper-container">
      <img ref="mobileCropperImage" :src="mobileCropImageUrl" />
    </div>
    <div class="cropper-actions">
      <button class="btn-cancel" @click="cancelCrop">取消</button>
      <button class="btn-confirm" @click="confirmCrop">确认裁剪并上传</button>
    </div>
  </div>
</div>
```

**添加的样式**：
- `.cropper-modal`：全屏半透明背景模态框
- `.cropper-modal-content`：白色内容区域，圆角阴影
- `.cropper-container`：裁剪器容器
- `.cropper-actions`：操作按钮区域
- `.btn-cancel`：取消按钮（灰色）
- `.btn-confirm`：确认按钮（渐变蓝色绿色）

**修改的上传提示**：
- 当选择图片类型时，显示"选择图片后将自动弹出裁剪窗口"
- 当选择视频类型时，显示"上传手机端背景"按钮（原逻辑）

### 3. Mobile端无需修改

Mobile端已经使用 `background-size: cover` 显示背景，裁剪后的图片直接适配手机屏幕，无需任何前端修改。

## 为什么这样做

1. **手机屏幕比例问题**：手机屏幕约6.67英寸，比例约9:20（1080x2400），如果上传横屏图片（如16:9），使用 `background-size: cover` 会导致上下被大幅裁剪，关键内容可能丢失。

2. **前端裁剪优势**：
   - 用户可以预览裁剪效果
   - 精确控制显示区域
   - 上传后直接是适配图片，mobile端无需额外处理
   - 不增加后端负担

3. **cropper.js 选择**：
   - 轻量级（~10KB gzipped）
   - 功能强大，支持缩放、旋转、拖动
   - 社区活跃，文档完善
   - 支持移动端触摸操作

4. **自动上传流程**：裁剪完成后自动上传，简化操作流程，避免用户忘记点击上传按钮。

## 产生的结果

- Admin端手机背景上传时，选择图片会自动弹出裁剪窗口
- 裁剪框固定为9:20比例（手机屏幕比例）
- 用户可以拖动图片选择显示区域
- 裁剪后的图片自动上传到服务器
- Mobile端背景完美适配手机屏幕，无变形、无黑边、无内容丢失
- 视频类型背景不受影响，仍使用原有上传逻辑

## 验证方案

1. 打开 admin 后台（http://localhost:5175）
2. 进入"屏幕设置"标签页
3. 在"手机端背景"区域，选择类型为"图片"
4. 选择一张横屏图片（如16:9比例）
5. 弹出裁剪窗口，显示9:20比例的裁剪框
6. 拖动图片选择想要显示的区域
7. 点击"确认裁剪并上传"
8. 上传成功提示
9. 打开手机端页面（http://localhost:5174）
10. 背景图片完美适配手机屏幕，显示裁剪选择的区域
