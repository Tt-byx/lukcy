# 项目测试方式和局域网访问说明

## 用户问题
用户想了解这个抽奖项目如何测试，以及电脑和手机连接同一个 WiFi 是否可以直接访问。

## 分析过程
1. 浏览了项目整体结构，确认这是一个包含 1 个 Spring Boot 后端 + 3 个 Vue 前端的抽奖系统
2. 检查了后端的 application.yml，确认端口为 8080，数据库为 MySQL，base-url 默认为 localhost
3. 检查了三个前端的 vite.config.js，确认都已配置 `host: '0.0.0.0'`，支持局域网访问
4. 检查了 CorsConfig 和 WebSocketConfig，确认跨域和 WebSocket 都已放开

## 所做工作
- 梳理了项目 4 个模块的端口和职责：
  - lucky-server (8080)：Spring Boot 后端
  - lucky-screen (5173)：大屏展示端
  - lucky-mobile (5174)：手机扫码端
  - lucky-admin (5175)：管理后台
- 确认三个前端 Vite dev server 已配置 `host: '0.0.0.0'`，局域网内可以直接访问
- 指出 base-url 配置需要改为局域网 IP，否则生成的二维码无法在手机上打开
- 给出了完整的测试流程指南

## 结果
- **手机连同一 WiFi 可以直接访问**，访问地址为 `http://电脑IP:5174`
- 需要注意将 `BASE_URL` 环境变量改为电脑的局域网 IP，确保二维码可用
- 给出了从数据库初始化到启动所有服务的完整测试步骤
