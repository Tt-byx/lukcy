# 20260520020000-Admin端用户管理功能

## 问题提示词

用户希望在 admin 后台新增"用户管理"tab，实现以下功能：
1. 查看在线用户
2. 查看参与抽奖用户
3. 对用户进行操作：禁言、移除直播间、移除抽奖资格

## 做了什么事情

### 1. WebSocket Handler 改造

**修改文件**：`lucky-server/.../websocket/LuckyWebSocketHandler.java`

**修改内容**：
- 替换 `Set<WebSocketSession>` 为三个 Map：
  - `sessionMap`: sessionId -> WebSocketSession
  - `sessionUserMap`: sessionId -> participantId
  - `userSessionMap`: participantId -> Set<sessionId>
- 在 `afterConnectionEstablished` 中解析 query 参数获取 participantId
- 在 `afterConnectionClosed` 中清理所有映射
- 新增方法：
  - `getOnlineParticipantIds()`：获取所有在线用户的 participantId 集合
  - `sendToUser()`：向指定用户的所有 session 推送消息
  - `disconnectUser()`：强制断开指定用户的所有连接

**关键代码**：
```java
private final ConcurrentHashMap<String, WebSocketSession> sessionMap = new ConcurrentHashMap<>();
private final ConcurrentHashMap<String, Long> sessionUserMap = new ConcurrentHashMap<>();
private final ConcurrentHashMap<Long, Set<String>> userSessionMap = new ConcurrentHashMap<>();

@Override
public void afterConnectionEstablished(WebSocketSession session) {
    sessionMap.put(session.getId(), session);
    // 解析 query 参数中的 participantId
    URI uri = session.getUri();
    if (uri != null && uri.getQuery() != null) {
        Map<String, String> params = parseQuery(uri.getQuery());
        String pidStr = params.get("participantId");
        if (pidStr != null) {
            Long participantId = Long.parseLong(pidStr);
            sessionUserMap.put(session.getId(), participantId);
            userSessionMap.computeIfAbsent(participantId, k -> ConcurrentHashMap.newKeySet())
                    .add(session.getId());
        }
    }
    broadcastOnlineCount();
}
```

### 2. 新建 AdminUserController

**新建文件**：`lucky-server/.../controller/AdminUserController.java`

**API 端点**：
- `GET /api/admin/user/list?activityId=`：获取指定活动所有参与者
- `GET /api/admin/user/online?activityId=`：获取当前在线用户
- `PUT /api/admin/user/{id}/mute?muted=`：禁言/取消禁言
- `PUT /api/admin/user/{id}/ban?banned=`：移出/恢复直播间
- `PUT /api/admin/user/{id}/lottery?removed=`：移除/恢复抽奖资格

**关键逻辑**：
- 在线用户：从 `webSocketHandler.getOnlineParticipantIds()` 获取 ID 集合，再从数据库查询详情
- 禁言：更新 `is_muted` 字段，WebSocket 通知用户
- 移出直播间：更新 `is_banned` 字段，通知用户并断开连接
- 移除抽奖：更新 `status` 字段（3=已移除），通知用户

**修改内容**：

if (dto.getParticipantId() != null) {

### 5. Admin 端添加用户管理 Tab

**修改文件**：`lucky-admin/src/App.vue`

**新增内容**：
1. Tab 按钮：在侧边栏添加"用户管理"
2. 用户管理面板：
   - 在线用户表格：姓名、学号、手机号、状态标签、操作按钮
   - 所有用户表格：支持搜索、显示在线/禁言/移出/无抽奖资格状态
3. 操作按钮：禁言、移出直播间、移除抽奖（带确认弹窗）
4. 切换 tab 时自动刷新数据
5. CSS 样式：表格、标签、按钮样式

## 为什么这样做

1. **WebSocket 身份映射**：原实现中 WebSocket session 是匿名的，无法关联到具体用户。通过解析连接时的 query 参数（Mobile 端已传递 participantId），建立 session-user 映射，实现按用户推送消息。

2. **三个 Map 设计**：
   - `sessionMap`：sessionId -> WebSocketSession，用于发送消息
   - `sessionUserMap`：sessionId -> participantId，用于断开连接时清理
   - `userSessionMap`：participantId -> Set<sessionId>，用于按用户查找 session（一个用户可能有多个 session）

3. **禁言检查**：在弹幕发送时检查用户状态，而不是在前端禁用输入框，因为用户可能通过其他方式绕过前端限制。

4. **移出直播间**：先发送通知消息再断开连接，让用户看到被移出的原因。

5. **Admin 端不连接 WebSocket**：在线列表通过 HTTP 请求获取（切换 tab 时刷新 + 手动刷新按钮），避免 Admin 端也占一个 session 的复杂度。

## 产生的结果

- Admin 端新增"用户管理"tab
- 可以查看在线用户列表（实时）
- 可以查看所有参与者列表（支持搜索）
- 可以对用户进行操作：
  - 禁言：用户无法发送弹幕，收到提示
  - 移出直播间：用户被踢回注册页
  - 移除抽奖资格：用户不参与后续抽奖
- Mobile 端收到操作通知后显示相应提示

## 状态字段说明

| 字段 | 值 | 含义 |
|------|-----|------|
| `status` | 1 | 正常参与 |
| `status` | 0 | 已中奖 |
| `status` | 3 | 已移除抽奖资格 |
| `is_muted` | 0/1 | 正常/禁言 |
| `is_banned` | 0/1 | 正常/已移出直播间 |
