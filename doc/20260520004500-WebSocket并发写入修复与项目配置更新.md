## 做了什么事情

### 1. WebSocket 并发写入修复

**问题诊断**：当多个 WebSocket 客户端同时连接时，`afterConnectionEstablished()` 会并发调用 `broadcastOnlineCount()`，而 `broadcast()` 方法会遍历所有 session 发送消息。如果多个线程同时向同一个 session 写入，会抛出 `IllegalStateException: The remote endpoint was in state [TEXT_PARTIAL_WRITING]`。

**修改文件**：`lucky-server/.../websocket/LuckyWebSocketHandler.java`

**修改内容**：
- 在 `broadcast()` 方法中，对每个 session 的 `sendMessage()` 调用添加 `synchronized (session)` 块
- 在异常捕获中添加 `IllegalStateException`（除了原有的 `IOException`）

**修改后代码**：
```java
public void broadcast(String type, Object data) {
    JSONObject msg = new JSONObject();
    msg.put("type", type);
    msg.put("data", data);
    String json = msg.toJSONString();

    TextMessage textMessage = new TextMessage(json);
    for (WebSocketSession session : sessions) {
        if (session.isOpen()) {
            synchronized (session) {
                try {
                    session.sendMessage(textMessage);
                } catch (IOException | IllegalStateException e) {
                    log.error("发送消息失败: {}", session.getId(), e);
                }
            }
        }
    }
}
```

## 为什么这样做

1. **synchronized 块**：WebSocket 的 `sendMessage()` 不是线程安全的。当多个客户端同时连接时，`afterConnectionEstablished()` 会在不同线程中调用，这些线程会并发执行 `broadcast()`，导致对同一个 session 的并发写入。`synchronized (session)` 确保同一时间只有一个线程能向特定 session 写入。

2. **捕获 IllegalStateException**：当并发写入发生时，除了 `IOException`，还可能抛出 `IllegalStateException`（表示 session 正在写入状态）。添加此异常捕获确保程序不会崩溃。

3. **配置更新**：用户要求将质量保证流程写入配置文件，确保未来修改代码时遵循"先验证再输出"的原则。
