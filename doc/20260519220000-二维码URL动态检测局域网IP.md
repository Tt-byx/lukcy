# 二维码URL动态检测局域网IP

## 提示词

用户反馈：手机电脑连接同一个WiFi，测试时screen大屏上生成的二维码跳转的是localhost地址，手机扫码无法访问。希望二维码指向实际的局域网地址。

## 做了什么

### 1. screen 二维码 URL 动态检测

修改 `lucky-screen/src\App.vue` 第 85 行，将 `MOBILE_URL` 的回退值从硬编码的 `http://localhost:5174` 改为动态检测：

```js
// 修改前
const MOBILE_URL = import.meta.env.VITE_MOBILE_URL || 'http://localhost:5174'

// 修改后
const MOBILE_URL = import.meta.env.VITE_MOBILE_URL || `${window.location.protocol}//${window.location.hostname}:5174`
```

### 2. 同步更新局域网 IP 配置

将旧 IP `172.27.31.152` 更新为 `172.27.108.225`：

- `lucky-mobile/.env.development` — `VITE_API_BASE`
- `lucky-server/src/main/resources/application.yml` — `lucky.base-url`

## 为什么这样做

之前二维码指向 `localhost:5174`，手机扫码后访问的是手机自身的 localhost，自然打不开。改用 `window.location.hostname` 后，URL 会自动跟随大屏页面的访问地址：

- 电脑通过 `http://192.168.1.5:5173` 打开大屏 → 二维码生成 `http://192.168.1.5:5174`
- 电脑通过 `http://localhost:5173` 打开大屏 → 二维码生成 `http://localhost:5174`（本机测试不受影响）

这样无论在哪个网络环境下，二维码都能正确指向移动端地址，不需要手动改 IP。

## 产生的结果

- 手机和电脑连同一 WiFi 时，手机扫码可直接访问移动端页面
- 本机测试（localhost）行为不变
- 生产环境仍可通过 `.env.production` 的 `VITE_MOBILE_URL` 环境变量覆盖

## 注意事项

- `lucky-mobile/.env.development` 中 `VITE_API_BASE` 写的是 `http://172.27.31.152:8080`，换网络后如果 IP 变了需要同步更新
- `lucky-mobile` 和 `lucky-screen` 的 vite.config.js 都已配置 `host: '0.0.0.0'`，确保局域网可访问
