## 做了些什么事情

### 1. 探索项目结构
- 查看了项目的根目录结构，识别出四个主要组件：
  - `lucky-server/` - 后端服务（Spring Boot）
  - `lucky-screen/` - 大屏幕端（Vue 3）
  - `lucky-mobile/` - 手机端（Vue 3）
  - `lucky-admin/` - 后端管理端（Vue 3）

### 2. 分析技术栈
**后端（lucky-server）：**
- Spring Boot 3.2.5 + Java 17
- MySQL 数据库
- MyBatis-Plus ORM 框架
- WebSocket 实时通信
- 阿里云 OSS（文件存储）
- FastJSON（JSON 处理）

**前端（三端通用）：**
- Vue 3.x + Vite 5.x
- WebSocket 实时通信

### 3. 查看核心代码
**数据库设计（7个核心表）：**
- activity（活动表）
- participant（参与者表）
- lottery_round（抽奖轮次表）
- winner（中奖记录表）
- danmaku（弹幕表）
- sensitive_word（敏感词表）
- screen_config（屏幕配置表）

**API 端点：**
- ActivityController - 活动管理
- ParticipantController - 参与者管理
- LotteryController - 抽奖管理
- DanmakuController - 弹幕管理
- ScreenConfigController - 屏幕配置

### 4. 分析功能模块
**大屏幕端功能：**
- 动态二维码展示
- 实时弹幕展示（全屏）
- 抽奖动画（滚动名字、中奖展示）
- 参与人数/在线人数统计
- 动态背景支持

**手机端功能：**
- 学号快速注册
- 实时弹幕发送
- 直播间体验
- 参与抽奖

**后端管理功能：**
- 活动生命周期管理
- 参与者管理（禁言/移除）
- 抽奖轮次管理
- 弹幕审核
- 屏幕配置自定义
- 历史活动查看

### 5. 了解部署架构
- 部署环境：阿里云学生机（2核2G内存、40G存储）
- 并发支持：2000 人规模
- WebSocket 连接管理：基于内存的 ConcurrentHashMap

## 项目特点总结

### ✅ 技术优势
1. **三端分离**：独立开发、独立部署、独立维护
2. **实时通信**：WebSocket 实现低延迟弹幕和抽奖状态推送
3. **高并发支持**：2000 人规模的实时互动能力
4. **灵活配置**：屏幕配置、弹幕样式、背景定制

### 🎯 业务价值
1. **互动性强**：扫码发弹幕、参与抽奖
2. **仪式感足**：大屏展示、滚动动画、中奖效果
3. **管理便捷**：后端统一管理活动、参与者、抽奖流程
4. **可扩展性**：支持多轮抽奖、历史活动归档

### 📊 技术指标
- **并发用户**：2000 人
- **实时延迟**：< 100ms（WebSocket）
- **存储方案**：MySQL（本地）+ 阿里云 OSS（文件）
- **前端框架**：Vue 3 + Vite
