# 20260519235500-修复二维码URL环境变量优先级问题

## 问题提示词

用户反馈：手机扫码还是跳转localhost，不是电脑的ip的5174端口

## 做了什么事情

### 问题诊断

检查代码发现：
1. `lucky-screen/src/App.vue` 第85行已正确配置动态hostname检测：
   ```js
   const MOBILE_URL = import.meta.env.VITE_MOBILE_URL || `${window.location.protocol}//${window.location.hostname}:5174`
   ```

2. 但 `lucky-screen/.env.development` 中设置了：
   ```
   VITE_MOBILE_URL=http://localhost:5174
   ```

### 根本原因

Vite 环境变量 `import.meta.env.VITE_MOBILE_URL` 的优先级高于代码中的 `||` 默认值。

当 `.env.development` 中定义了 `VITE_MOBILE_URL=http://localhost:5174` 时，代码中的动态 hostname 检测永远不会生效。

### 修复方案

删除 `.env.development` 中的 `VITE_MOBILE_URL` 配置，让代码使用动态检测：

**修改文件：** `lucky-screen/.env.development`

```diff
# 本地开发环境配置（npm run dev 时生效）
# 局域网测试：用你电脑的 IP，手机连同一 WiFi 即可访问
VITE_API_BASE=http://localhost:8080
VITE_WS_URL=ws://localhost:8080/ws/lucky
-VITE_MOBILE_URL=http://localhost:5174
+# VITE_MOBILE_URL 已移除，使用代码中的动态 hostname 检测
```

## 为什么这样做

使用 `window.location.hostname` 动态检测有以下优势：
1. **自动适应**：无论在什么网络环境下，都会自动使用当前访问的主机名
2. **无需手动配置**：不需要每次换网络都修改 IP 地址
3. **局域网友好**：手机通过电脑 IP 访问时，二维码自动指向同一 IP 的 5174 端口

## 产生的结果

- 二维码 URL 现在会动态使用当前访问的 hostname
- 手机通过 `http://172.27.108.225:5173` 访问大屏页面时，二维码会指向 `http://172.27.108.225:5174`
- **需要重启 lucky-screen 开发服务器**才能生效

## 操作步骤

1. 停止当前运行的 `lucky-screen` 开发服务器
2. 重新运行 `npm run dev`
3. 通过电脑 IP 访问大屏页面（如 `http://172.27.108.225:5173`）
4. 扫描二维码，应该会跳转到 `http://172.27.108.225:5174`
