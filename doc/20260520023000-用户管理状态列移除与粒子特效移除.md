# 20260520023000-用户管理状态列移除与粒子特效移除

## 问题提示词

用户提出三个需求：
1. 用户管理里面，在线用户可以有状态，但是所有参与者没必要有状态列，删了吧
2. Screen 屏幕上的粒子特效不要，看着太傻了
3. 弹幕滚动动画在大量弹幕时会不会卡

## 做了什么事情

### 1. 删除"所有参与者"表格中的状态标签

**修改文件**：`lucky-admin/src/App.vue`

**修改内容**：
- 删除了"所有参与者"表格中状态标签所在的 `<td>`（包含在线、禁言、已移出、无抽奖资格、已中奖等标签）
- 保留了操作按钮列（禁言、移出直播间、移除抽奖）
- "在线用户"表格的状态列不受影响，保持原样

删除前表格有 5 列：姓名、学号、手机号、状态标签、操作按钮
删除后表格有 4 列：姓名、学号、手机号、操作按钮

### 2. 删除 Screen 页面粒子特效

**修改文件**：`lucky-screen/src/App.vue`

**修改内容**：
- 模板：删除了 `.bg-particles` 容器及 30 个 `.particle` 元素
- JS：删除了 `particleStyle()` 函数（该函数为每个粒子随机生成大小、位置、动画延迟和时长）
- CSS：删除了 `.bg-particles`、`.particle` 样式和 `@keyframes float` 动画

### 3. 弹幕性能分析

**结论**：正常使用场景（每秒 1-3 条弹幕）不会卡。

当前弹幕实现的特点：
- 每条弹幕是一个独立 DOM 元素 + CSS 动画（`danmaku-fly`）
- 使用 `animationend` 事件自动清理飞出屏幕的弹幕
- 弹幕速度有 ±30% 随机波动，避免齐头并进

潜在风险：如果瞬间涌入大量弹幕（如每秒 10+ 条），可能因 DOM 节点过多导致性能问题。建议后续如遇到高并发场景，可增加弹幕队列限流（如最多同时显示 50 条）。

## 为什么这样做

### 需求 1：删除状态标签
"所有参与者"表格展示的是全部用户，状态标签（在线、禁言、已移出等）信息量大但视觉噪音也大。用户认为这些状态信息在"在线用户"表格中已经可以看到，所有参与者表格只需保留基本操作即可。

### 需求 2：删除粒子特效
用户觉得粒子漂浮效果"看着太傻了"，影响大屏展示的专业感。删除后背景更简洁干净。

## 产生的结果

- Admin 后台"用户管理" → "所有参与者"表格中不再显示状态标签，只保留姓名、学号、手机号和操作按钮
- Screen 大屏页面不再有白色粒子漂浮背景效果，背景更简洁
- 两个项目（lucky-admin、lucky-screen）构建均通过，无报错
