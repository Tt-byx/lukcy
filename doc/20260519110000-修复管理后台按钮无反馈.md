# 修复管理后台所有按钮无反馈的问题

## 用户问题
管理后台的"创建活动"按钮点击没反应，没有 API 请求，没有报错。用户怀疑后端接口没实现。

## 分析过程
1. 检查后端 `ActivityController.java`：`POST /api/activity?name=xxx` 接口**已实现**，代码正确
2. 检查前端 `lucky-admin/src/App.vue` 的 `createActivity()` 函数，发现两个问题：
   - 输入框为空时：`if (!newActivityName.value.trim()) return;` — **静默返回，零反馈**
   - 后端返回非200时：`if (data.code === 200) { ... }` — **没有 else 分支，静默失败**
3. 进一步排查发现**所有增删改操作**都有同样的"静默失败"模式：
   - `createActivity` — 空输入静默返回 + 非200无反馈
   - `updateActivityStatus` — 不检查响应码直接改本地状态
   - `createRound` — 空输入静默返回 + 非200无反馈
   - `reviewDanmaku` — 不检查响应码直接更新本地
   - `addSensitiveWord` — 空输入静默返回 + 非200无反馈
   - `deleteSensitiveWord` — 不检查响应码直接移除

## 修改内容

### lucky-admin/src/App.vue — 6 个函数全部修复

修复模式（以 createActivity 为例）：
```javascript
// 修复前
async function createActivity() {
  if (!newActivityName.value.trim()) return  // 静默返回
  try {
    const res = await fetch(...)
    const data = await res.json()
    if (data.code === 200) { /* 成功 */ }
    // 失败静默
  } catch (e) {
    alert('创建失败')  // 信息不够
  }
}

// 修复后
async function createActivity() {
  if (!newActivityName.value.trim()) {
    alert('请输入活动名称')  // ★ 空输入有提示
    return
  }
  try {
    const res = await fetch(...)
    const data = await res.json()
    if (data.code === 200) {
      /* 成功 */
    } else {
      alert(data.message || '创建失败')  // ★ 后端错误有提示
    }
  } catch (e) {
    console.error('创建活动失败:', e)  // ★ 控制台可见
    alert('创建失败：' + e.message)  // ★ 显示具体错误
  }
}
```

具体修改的函数：
| 函数 | 修复内容 |
|------|----------|
| `createActivity` | 空输入提示 + else 错误提示 + console.error + 成功后刷新数据 |
| `updateActivityStatus` | 检查响应码 + else 错误提示 + console.error |
| `createRound` | 空输入提示 + else 错误提示 + console.error |
| `reviewDanmaku` | 检查响应码 + else 错误提示 + console.error |
| `addSensitiveWord` | 空输入提示 + 检查响应码 + else 错误提示 + console.error |
| `deleteSensitiveWord` | 检查响应码 + else 错误提示 + console.error |

## 结果
- 后端 API 是完整实现的，没有问题
- 前端所有按钮现在都会给出明确反馈：输入为空会提示、接口失败会提示具体错误信息
- 控制台也会打印错误日志，方便调试
