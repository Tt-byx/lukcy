# 抽奖程序项目搭建

## 本次问答的提示词

用户请求做一个抽奖程序，需求如下：
- 大屏幕展现界面，显示二维码
- 用户用微信扫码填写信息参与
- 用户可以发弹幕，弹幕展现在大屏幕
- 后台点击抽奖，播放动画，抽取幸运观众
- 支持多轮抽奖
- 弹幕需要关键词过滤

经过多轮沟通确认：
- 技术栈：Spring Boot + Vue 3
- 微信接入：扫码跳转普通网页
- 用户信息：姓名、手机号、学号
- 规模：2000人左右
- 服务器：阿里云 2核4G
- 数据库：MySQL
- 弹幕：关键词屏蔽 + 管理员审核

## 做了些什么事情

### 1. 设计系统架构
- 三端分离：大屏展示、用户扫码页、管理后台
- 后端提供 REST API + WebSocket 实时通信
- 数据库设计6张表：活动、参与者、抽奖轮次、中奖记录、弹幕、敏感词

### 2. 搭建 Spring Boot 后端
创建了完整的后端项目结构：
- `lucky-server/pom.xml` - Maven 项目配置，依赖 Spring Boot 3.2.5、MyBatis-Plus、WebSocket、MySQL
- `LuckyApplication.java` - 启动类
- `entity/` - 6个实体类：Activity、Participant、LotteryRound、Winner、Danmaku、SensitiveWord
- `mapper/` - 6个 MyBatis-Plus Mapper 接口
- `dto/` - 数据传输对象：Result、ParticipantDTO、DanmakuDTO、LotteryDTO
- `service/` - 业务逻辑层：
  - ActivityService - 活动管理
  - ParticipantService - 参与者注册，去重校验
  - DanmakuService - 弹幕发送，敏感词过滤，自动审核
  - LotteryService - 抽奖逻辑，随机算法，多轮支持
  - SensitiveWordService - 敏感词管理，内容过滤
- `controller/` - REST 接口：
  - ActivityController - 活动 CRUD
  - ParticipantController - 参与者注册、查询
  - DanmakuController - 弹幕发送、审核
  - LotteryController - 轮次管理、触发抽奖
  - SensitiveWordController - 敏感词增删
  - WsController - WebSocket 在线人数
- `websocket/LuckyWebSocketHandler.java` - WebSocket 处理器，支持广播弹幕、抽奖通知
- `config/` - CORS 跨域配置、WebSocket 配置

### 3. 创建数据库脚本
`schema.sql` 包含：
- 建库语句
- 6张表的建表语句（含索引）
- 默认敏感词数据

### 4. 开发大屏展示页面 (lucky-screen)
- Vue 3 + Vite 项目
- 二维码生成（qrcode.js）
- 弹幕飘过动画（CSS3 animation）
- 抽奖滚动动画
- 中奖结果展示（头像卡片 + 弹出动画）
- 背景粒子效果
- WebSocket 实时接收弹幕和抽奖通知

### 5. 开发用户扫码页面 (lucky-mobile)
- 移动端适配
- 注册表单（姓名、学号、手机号）
- 弹幕发送界面
- 快捷弹幕（10条预设）
- 发送记录查看
- localStorage 保存注册状态

### 6. 开发管理后台 (lucky-admin)
- 侧边栏导航
- 活动管理：创建活动、开始/结束活动
- 抽奖管理：创建轮次、触发抽奖、查看中奖记录
- 弹幕审核：查看待审核弹幕、通过/拒绝
- 敏感词管理：添加/删除敏感词

### 7. 创建启动脚本
- `start.sh` - Linux/Mac 启动脚本
- `start.bat` - Windows 启动脚本

## 为什么这样做

1. **前后端分离**：大屏、手机端、管理后台三个独立 Vue 项目，可以独立部署，互不影响
2. **WebSocket 实时通信**：弹幕和抽奖需要实时推送到大屏，WebSocket 是最佳选择
3. **敏感词过滤**：在弹幕发送时自动检查，命中敏感词直接拒绝，减少人工审核压力
4. **多轮抽奖**：通过 lottery_round 表支持多轮，每轮独立记录中奖者
5. **参与者去重**：通过 studentId + activityId 联合唯一检查，防止重复参与
6. **CSS3 动画**：弹幕飘过、抽奖滚动、中奖弹出都用纯 CSS 实现，性能好

### 后端 API 接口
| 模块 | 接口 | 说明 |
|------|------|------|
| 活动 | POST /api/activity | 创建活动 |
| 活动 | GET /api/activity/current | 获取当前活动 |
| 活动 | PUT /api/activity/{id}/status | 更新活动状态 |
| 参与者 | POST /api/participant/register | 注册参与 |
| 参与者 | GET /api/participant/list | 参与者列表 |
| 参与者 | GET /api/participant/count | 参与人数 |
| 弹幕 | POST /api/danmaku/send | 发送弹幕 |
| 弹幕 | PUT /api/danmaku/{id}/review | 审核弹幕 |
| 弹幕 | GET /api/danmaku/pending | 待审核弹幕 |
| 抽奖 | POST /api/lottery/round | 创建轮次 |
| 抽奖 | POST /api/lottery/draw/{roundId} | 触发抽奖 |
| 敏感词 | GET/POST/DELETE /api/sensitive-word | 敏感词 CRUD |
| WebSocket | ws://localhost:8080/ws/lucky | 实时通信 |
