## 三、问题修复

### 3.1 上传失败问题

**问题原因**：
1. `screen_config` 表未自动创建，导致数据库查询失败（500错误）
2. Spring Boot 默认文件上传大小限制为1MB

**修复方案**：
1. 新增 `DatabaseInitializer` 类，启动时自动创建表和默认配置
2. 在 `application.yml` 中配置 `max-file-size: 100MB` 和 `max-request-size: 100MB`

### 3.2 存储方式切换

**从本地存储切换到OSS存储**：
1. 后端文件上传改为上传到阿里云OSS
2. 返回的URL为OSS绝对路径（`https://...`）
3. 前后端代码同步调整，不再拼接API_BASE前缀

## 四、使用说明

### 4.1 配置步骤

1. 设置阿里云OSS环境变量：
```powershell
$env:OSS_ACCESS_KEY_ID="your-access-key-id"
$env:OSS_ACCESS_KEY_SECRET="your-access-key-secret"
```

2. 启动后端服务：
```powershell
cd lucky-server
mvn spring-boot:run
```

3. 启动管理后台：
```powershell
cd lucky-admin
npm install
npm run dev
```

4. 启动大屏页面：
```powershell
cd lucky-screen
npm install
npm run dev
```

### 4.2 操作流程

1. 打开管理后台（默认 http://localhost:5175）
2. 点击侧边栏「屏幕设置」
3. 选择背景类型（图片/视频）
4. 选择文件并点击「上传背景」
5. 大屏页面自动加载新背景（或等待30秒自动刷新）
6. 如需恢复默认背景，点击「清除自定义背景」
