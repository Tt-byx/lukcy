# 阿里云服务器部署指南

## 提示词
用户需要将抽奖程序部署到阿里云服务器（IP: 101.201.233.194），要求：
- 前端和nginx打包成一个镜像
- MySQL数据打包成镜像
- 不使用挂载
- 端口：8080、5173、5174、5175

---

## 做了些什么事情

### 1. 修改前端配置文件

#### 1.1 修改 mobile 生产环境配置
**文件：** `lucky-mobile/.env.production`
```env
VITE_API_BASE=/api
VITE_WS_URL=ws://101.201.233.194:5174/ws/lucky
VITE_MOBILE_URL=http://101.201.233.194:5174
```

#### 1.2 修改 screen 生产环境配置
**文件：** `lucky-screen/.env.production`
```env
VITE_API_BASE=/api
VITE_WS_URL=ws://101.201.233.194:5173/ws/lucky
```

### 2. 修改 Nginx 配置

**文件：** `docker/nginx/nginx.conf`

配置三个独立端口的server：
- 端口 5173：大屏展示
- 端口 5174：手机端
- 端口 5175：管理后台

每个server都配置了API代理和WebSocket代理。

### 3. 创建 MySQL Dockerfile

**文件：** `docker/Dockerfile.mysql`
```dockerfile
FROM mysql:8.0
COPY lucky-server/src/main/resources/schema.sql /docker-entrypoint-initdb.d/init.sql
CMD ["--character-set-server=utf8mb4", "--collation-server=utf8mb4_unicode_ci"]
```

### 4. 修改 docker-compose.yml

**文件：** `docker-compose.yml`

配置三个服务：
- mysql：使用自定义Dockerfile，打包数据库初始化脚本
- backend：后端服务，端口8080
- frontend：前端服务，端口5173、5174、5175

### 5. 创建服务器端 docker-compose

**文件：** `docker-compose.prod.yml`

服务器端配置，不需要build，直接使用镜像。

### 6. 创建构建脚本

**文件：** `docker-build.sh`

自动化构建和打包脚本。

## 为什么这样做

### 1. **打包成镜像的优势**
- 部署简单，只需加载镜像并启动
- 环境一致，避免服务器环境差异
- 数据库初始化脚本打包到镜像中，无需手动执行

### 2. **端口分配**
- 5173：大屏展示（扫码展示）
- 5174：手机端（用户访问）
- 5175：管理后台（管理员使用）
- 8080：后端API

### 3. **Nginx配置**
- 每个前端独立端口，避免路径冲突
- 统一的API代理配置
- WebSocket代理支持

## 部署步骤

### 步骤1：上传文件到服务器

将以下文件上传到服务器：
- `lucky-docker.tar.gz`（338MB）
- `docker-compose.prod.yml`

**上传方式：**
```bash
# 方式1：使用scp命令
scp lucky-docker.tar.gz root@101.201.233.194:/root/
scp docker-compose.prod.yml root@101.201.233.194:/root/

# 方式2：使用FTP工具（如FileZilla）
```

### 步骤2：在服务器上解压和加载镜像

```bash
# 1. 解压文件
tar -xzf lucky-docker.tar.gz

# 2. 加载Docker镜像
docker load -i lucky-images.tar

# 3. 查看加载的镜像
docker images | grep lucky
```

**预期输出：**
```
lucky-frontend   latest   xxxxx   10 seconds ago   180MB
lucky-backend    latest   xxxxx   2 minutes ago    450MB
lucky-mysql      latest   xxxxx   3 minutes ago    520MB
```

### 步骤3：启动服务

```bash
# 1. 重命名配置文件
mv docker-compose.prod.yml docker-compose.yml

# 2. 启动所有服务
docker-compose up -d

# 3. 查看服务状态
docker-compose ps
```

**预期输出：**
```
NAME            IMAGE            STATUS          PORTS
lucky-backend   lucky-backend    Up 2 minutes    0.0.0.0:8080->8080/tcp
lucky-frontend  lucky-frontend   Up 2 minutes    0.0.0.0:5173->5173/tcp, 0.0.0.0:5174->5174/tcp, 0.0.0.0:5175->5175/tcp
lucky-mysql     lucky-mysql      Up 2 minutes    0.0.0.0:3306->3306/tcp
```

### 步骤4：验证服务

```bash
# 1. 检查后端API
curl http://localhost:8080/api/activity/current

# 2. 检查前端页面
curl http://localhost:5174/

# 3. 查看日志
docker-compose logs -f
```

### 步骤5：开放端口（阿里云安全组）

在阿里云控制台开放以下端口：
- 8080（后端API）
- 5173（大屏展示）
- 5174（手机端）
- 5175（管理后台）
- 3306（MySQL，可选，如果需要远程连接）

**操作路径：**
1. 登录阿里云控制台
2. 进入ECS实例
3. 点击"安全组"
4. 添加入站规则：
   - 协议：TCP
   - 端口：8080/5173/5174/5175
   - 授权对象：0.0.0.0/0

## 常见问题

### Q1：MySQL启动失败
**症状：** `lucky-mysql` 容器不断重启
**解决：**
```bash
# 查看日志
docker-compose logs mysql

# 如果是权限问题，删除数据卷重新启动
docker-compose down -v
docker-compose up -d
```

### Q2：后端连接数据库失败
**症状：** `lucky-backend` 启动后报数据库连接错误
**解决：**
```bash
# 检查MySQL是否健康
docker-compose ps

# 等待MySQL完全启动（约30秒）
sleep 30
docker-compose restart backend
```

### Q3：前端页面空白
**症状：** 访问5173/5174/5175显示空白
**解决：**
```bash
# 检查nginx配置
docker exec lucky-frontend nginx -t

# 查看nginx日志
docker-compose logs frontend
```

### Q4：WebSocket连接失败
**症状：** 弹幕功能不可用
**解决：**
```bash
# 检查防火墙是否开放WebSocket端口
# 检查nginx配置中的WebSocket代理

# 测试WebSocket连接
curl -i -N -H "Connection: Upgrade" -H "Upgrade: websocket" http://localhost:5174/ws/lucky
```
