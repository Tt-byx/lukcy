# 20260520153000-Mobile界面精简与人数同步与弹幕审核删除

## 问题提示词

用户提出三个需求：
1. mobile 界面我不想要上面显示的抽奖活动名字和在线人数，也不要那个大阴影
2. screen 界面的参与人数和在线人数和后台的人数对不上
3. 弹幕审核的 tab 我觉得没啥用删了吧

## 做了什么事情

### 1. Mobile 界面精简

**修改文件**：`lucky-mobile/src/App.vue`

**修改内容**：
- 模板：删除了 `<header class="live-header">` 整个元素（包含活动名字和在线人数显示）
- JS：删除了 `onlineCount` ref 及 WebSocket 中 `online_update` 消息的处理逻辑
- CSS：删除了 `.live-header`（半透明黑色遮罩 + 毛玻璃模糊）、`.mobile-app::before`（全局径向渐变光晕）、`.activity-name`、`.online-badge` 样式

保留了 `activityName` ref 和 `fetchActivity()`，因为注册页面（logo 区域）仍需要显示活动名字。

### 2. Screen 人数与后台同步

**修改文件**：`lucky-server/src/main/java/com/lucky/websocket/LuckyWebSocketHandler.java`

**问题根因**：后端有两种在线人数计算方式：
- `getOnlineCount()`：统计 WebSocket session 数量（同一用户多设备登录会重复计数）
- `getOnlineParticipantIds()`：统计去重的用户数

Screen 大屏通过 `broadcastOnlineCount()` 调用的是 `getOnlineCount()`（session 数量），而 Admin 后台调用的是 `getOnlineParticipantIds()`（去重用户数），导致两边数字不一致。

**修改内容**：将 `broadcastOnlineCount()` 方法中的 `getOnlineCount()` 改为 `getOnlineParticipantIds().size()`，使广播的在线人数与 Admin 后台一致（按去重用户计算）。

### 3. 删除弹幕审核 Tab

**修改文件**：`lucky-admin/src/App.vue`

**修改内容**：
- 模板：删除了侧边栏中的"弹幕审核"链接（第 11 行）
- 模板：删除了弹幕审核内容区（包含待审核列表、通过/拒绝按钮）
- JS：删除了 `pendingDanmakus` ref、`fetchPendingDanmakus()` 函数、`reviewDanmaku()` 函数
- JS：删除了 `watch` 和 `onMounted` 中 `tab === 'danmaku'` 的分支
- JS：修改了 `currentTab` 初始化逻辑，如果 localStorage 中保存的 tab 是 `'danmaku'` 则回退到 `'activity'`
- CSS：删除了 `.danmaku-review-item`、`.danmaku-content`、`.review-btns`、`.btn-approve`、`.btn-reject` 样式

## 为什么这样做

### 需求 1：Mobile 界面精简
用户觉得手机端顶部的活动名字和在线人数显示多余，大阴影效果影响视觉体验。删除后直播间界面更简洁，弹幕消息区域更突出。

### 需求 2：人数同步
Screen 大屏和 Admin 后台使用了不同的在线人数计算口径。session 数量会在同一用户多设备登录时重复计数，而去重用户数更准确地反映实际在线人数。统一使用去重用户数后，两个页面显示一致。

### 需求 3：删除弹幕审核
用户认为弹幕审核功能没有实际使用价值，删除后侧边栏更简洁。

## 产生的结果

- Mobile 手机端进入直播间后，顶部不再显示活动名字和在线人数，无毛玻璃阴影和径向渐变光晕效果
- Screen 大屏的在线人数与 Admin 后台一致（按去重用户计算，不再因多设备登录而重复计数）
- Admin 后台侧边栏不再显示"弹幕审核"tab，如果之前保存的 tab 是弹幕审核会自动回退到活动管理
- 三个项目（lucky-mobile、lucky-server、lucky-admin）构建均通过，无报错
