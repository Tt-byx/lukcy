#### 1.2 创建Redis配置类

- 配置RedisTemplate
- 使用Jackson2JsonRedisSerializer序列化
- 支持存储复杂对象

#### 1.3 创建Redis工具类
**文件：** `lucky-server/src/main/java/com/lucky/util/RedisUtil.java`
- 封装常用Redis操作
- 支持String、Set等数据结构
- 支持过期时间设置

#### 1.4 修改application.yml
**文件：** `lucky-server/src/main/resources/application.yml`
```yaml
spring:
  data:
    redis:
      host: ${REDIS_HOST:localhost}
      port: ${REDIS_PORT:6379}
      password: ${REDIS_PASSWORD:}
      database: 0
      timeout: 3000ms
      lettuce:
        pool:
          max-active: 8
          max-wait: -1ms
          max-idle: 8
          min-idle: 0
```

#### 1.5 修改WebSocketHandler
**文件：** `lucky-server/src/main/java/com/lucky/websocket/LuckyWebSocketHandler.java`
- 在线用户信息存储到Redis
- 支持分布式部署
- 使用Redis Set存储在线用户ID
- 使用Redis计数器统计在线人数

#### 1.6 修改敏感词缓存
**文件：** `lucky-server/src/main/java/com/lucky/service/impl/SensitiveWordServiceImpl.java`
- 使用Redis缓存敏感词
- 设置1小时过期时间
- 增删改时自动清除缓存

#### 1.7 Docker Compose添加Redis
**文件：** `docker-compose.yml` 和 `docker-compose.prod.yml`
```yaml
redis:
  image: redis:7-alpine
  container_name: lucky-redis
  ports:
    - "6379:6379"
  command: redis-server --appendonly yes
  volumes:
    - redis-data:/data
  healthcheck:
    test: ["CMD", "redis-cli", "ping"]
    interval: 10s
    timeout: 5s
    retries: 5
  restart: unless-stopped
```

### 6. 第三优先级 - Nginx缓存/压缩

#### 6.1 Gzip压缩配置
**文件：** `docker/nginx/nginx.conf`
```nginx
# Gzip压缩配置
gzip on;
gzip_vary on;
gzip_proxied any;
gzip_comp_level 6;
gzip_buffers 16 8k;
gzip_http_version 1.1;
gzip_min_length 256;
gzip_types
    text/plain
    text/css
    text/xml
    text/javascript
    application/json
    application/javascript
    application/x-javascript
    application/xml
    application/xml+rss
    application/vnd.ms-fontobject
    application/x-font-ttf
    font/opentype
    image/svg+xml
    image/x-icon;
```

#### 6.2 静态资源缓存策略
```nginx
# 静态资源缓存策略
location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
    root /usr/share/nginx/html/screen;
    expires 7d;
    add_header Cache-Control "public, immutable";
    access_log off;
}
```

**缓存策略说明：**
- 静态资源（JS/CSS/图片/字体）缓存7天
- 添加Cache-Control: public, immutable
- 关闭静态资源访问日志

#### 6.3 代理头优化
```nginx
proxy_set_header X-Forwarded-Proto $scheme;
```

## 为什么这样做

### 1. **Redis**
- **分布式支持**：支持多实例部署，WebSocket在线用户信息共享
- **缓存优化**：敏感词缓存、数据缓存
- **计数器**：在线人数统计
- **面试加分**：Redis是后端面试必问技术点

### 2. **参数校验**
- **数据安全**：防止无效数据进入系统
- **代码规范**：使用标准JSR-303注解
- **用户体验**：返回明确的错误提示
- **面试加分**：体现代码规范意识

### 3. **Swagger**
- **API文档**：自动生成接口文档
- **调试方便**：可以直接在页面测试接口
- **团队协作**：前后端对接更方便
- **面试加分**：可以现场展示API文档

### 4. **Spring Security + JWT**
- **安全保障**：管理后台需要登录才能访问
- **无状态认证**：JWT Token，支持分布式
- **权限控制**：基于角色的访问控制
- **面试加分**：安全是后端最重要的能力之一

### 5. **DTO/VO转换**
- **分层规范**：Entity不直接暴露给前端
- **数据安全**：隐藏敏感字段（如手机号）
- **灵活性**：可以添加额外字段（如状态文本）
- **面试加分**：体现分层设计思想

### 6. **Nginx缓存/压缩**
- **性能优化**：减少传输体积，加快加载速度
- **带宽节省**：静态资源缓存，减少重复请求
- **用户体验**：页面加载更快
- **面试加分**：体现性能优化意识

## 面试话术

### 1. Redis
> "项目使用Redis实现了分布式在线用户管理，WebSocket连接时将用户ID存入Redis Set，断开时移除。使用Redis计数器实时统计在线人数。敏感词缓存也使用Redis，设置1小时过期，增删改时自动清除缓存。"

### 2. Spring Security + JWT
> "管理后台使用Spring Security + JWT进行认证授权。登录时验证用户名密码，生成JWT Token返回给前端。前端每次请求在Header中携带Token，后端通过JWT过滤器验证Token有效性。公开接口（如活动查询、弹幕发送）不需要认证，管理接口需要认证。"

### 3. 参数校验
> "使用JSR-303标准注解进行参数校验，如@NotBlank、@NotNull、@Size等。Controller方法参数添加@Valid注解，校验失败时由全局异常处理器返回友好的错误提示。"

### 4. DTO/VO转换
> "使用VO（View Object）模式，Entity不直接暴露给前端。VO转换工具类负责将Entity转换为VO，同时可以添加额外字段（如状态文本）、隐藏敏感字段（如手机号）。"

### 5. Nginx优化
> "Nginx配置了Gzip压缩，减少传输体积约60%-70%。静态资源设置7天缓存，添加Cache-Control: public, immutable头，减少重复请求。"
