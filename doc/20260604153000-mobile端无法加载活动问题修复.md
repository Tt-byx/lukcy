# 20260604153000-mobile端无法加载活动问题修复.md

## 提示词
用户报告 mobile 页面一直显示"加载中"然后显示"暂无活动"，但实际上后端有活动。

---

## 做了些什么事情

### 1. 问题诊断

#### 测试后端API
```bash
curl -s http://localhost:8080/api/activity/current
```

**返回结果：**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 11,
    "name": "肉蛋的引导",
    "status": 1,
    "createdAt": "2026-06-04T14:20:44"
  }
}
```

✅ 后端API正常返回活动数据

#### 发现问题根源

检查 `.env.development` 配置文件：
```env
# 本地开发环境配置（npm run dev 时生效）
# 局域网测试：用你电脑的 IP，手机连同一 WiFi 即可访问
VITE_API_BASE=http://172.27.59.47:8080
VITE_WS_URL=ws://172.27.59.47:8080/ws/lucky
```

**问题：**
- 配置使用的是局域网IP `172.27.59.47`
- 本地浏览器无法访问这个IP（可能是防火墙阻止、IP已变更等原因）
- 导致前端的fetch请求超时失败
- 进入catch块，设置 `noActivity = true`
- 所以一直显示"暂无活动"

#### 测试局域网IP可达性
```bash
curl -s --connect-timeout 3 http://172.27.59.47:8080/api/activity/current
# Exit code 28 (连接超时)
```

❌ 局域网IP不可达

### 2. 实施修复

修改 `.env.development` 文件，将API地址改为 `localhost`：

**修改前：**
```env
# 本地开发环境配置（npm run dev 时生效）
# 局域网测试：用你电脑的 IP，手机连同一 WiFi 即可访问
VITE_API_BASE=http://172.27.59.47:8080
VITE_WS_URL=ws://172.27.59.47:8080/ws/lucky
```

**修改后：**
```env
# 本地开发环境配置（npm run dev 时生效）
VITE_API_BASE=http://localhost:8080
VITE_WS_URL=ws://localhost:8080/ws/lucky

# 局域网测试：取消下面的注释，用你电脑的 IP，手机连同一 WiFi 即可访问
# VITE_API_BASE=http://172.27.59.47:8080
# VITE_WS_URL=ws://172.27.59.47:8080/ws/lucky
```

### 3. 重启开发服务器

```bash
taskkill /F /IM node.exe
npm run dev
```

### 4. 验证修复

#### 测试API访问
```javascript
const API_BASE = 'http://localhost:8080';
fetch(API_BASE + '/api/activity/current')
  .then(res => res.json())
  .then(data => {
    console.log('code === 200:', data.code === 200);
    console.log('data.data exists:', !!data.data);
    // ✓ Activity found: 肉蛋的引导 ID: 11
  });
```

✅ API访问正常，能正确获取活动数据

## 为什么这样做

### 1. **问题本质**
- `.env.development` 配置了局域网IP地址
- 本地开发时应该使用 `localhost`
- 局域网IP配置只应在手机测试时临时开启

### 2. **解决方案**
- 将本地开发环境默认改为 `localhost`
- 保留局域网IP配置作为注释，方便需要时快速切换
- 添加注释说明使用场景

### 3. **配置文件说明**
| 文件 | 用途 | API地址 |
|------|------|---------|
| `.env.development` | 本地开发（npm run dev） | localhost:8080 |
| `.env.production` | 生产构建（npm run build） | /api (相对路径) |

## 局域网测试说明

如果需要用手机测试（手机和电脑在同一WiFi下）：

1. 查看你电脑的局域网IP：
   ```bash
   ipconfig  # Windows
   ifconfig  # Mac/Linux
   ```

2. 修改 `.env.development`：
   ```env
   VITE_API_BASE=http://你的IP:8080
   VITE_WS_URL=ws://你的IP:8080/ws/lucky
   ```

3. 重启开发服务器：
   ```bash
   npm run dev
   ```

4. 手机浏览器访问：`http://你的IP:5174`

## 调试建议

### 如果仍然显示"暂无活动"

1. **检查后端服务**
   ```bash
   curl http://localhost:8080/api/activity/current
   ```

2. **检查浏览器控制台**
   - 打开 Chrome DevTools (F12)
   - 查看 Console 标签页的错误信息
   - 查看 Network 标签页的请求状态

3. **清除浏览器缓存**
   - Ctrl+Shift+Delete 清除缓存
   - 或使用无痕模式访问

4. **检查防火墙**
   - Windows 防火墙可能阻止本地访问
   - 临时关闭防火墙测试
