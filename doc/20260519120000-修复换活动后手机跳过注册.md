# 修复：换活动后手机端跳过注册、参与人数不更新

## 用户问题
1. 之前创建了活动A，手机扫码注册参与过
2. 经过代码修改后，原来的活动A没了
3. 重新创建活动B，手机扫码后发现：参与人数没变，但可以在里面发弹幕

## 分析过程

### 追踪代码执行流程
查看 `lucky-mobile/src/App.vue` 的 `onMounted` 逻辑：

```javascript
onMounted(() => {
  fetchActivity()          // ← 异步，去拿活动B的ID
  const saved = localStorage.getItem('lucky_participant')  // ← 读到活动A的数据！
  if (saved) {
    participant.value = JSON.parse(saved)
    registered.value = true  // ← 立即设为 true！
  }
})
```

### 问题根因
1. `fetchActivity()` 是**异步**的，还没返回时 localStorage 已经读了
2. localStorage 里存的是**旧活动A**的参与者数据
3. `registered = true` → 页面直接显示弹幕界面，**跳过了注册表单**
4. 用户以为已经参与了，实际上对**活动B**根本没注册
5. 所以参与人数当然不变
6. 弹幕能发送是因为弹幕接口没校验 `participantId` 是否属于当前活动

## 修改内容

### lucky-mobile/src/App.vue — `onMounted` 增加活动校验

```javascript
onMounted(() => {
  // 先读取 localStorage（快速显示已有状态）
  const saved = localStorage.getItem('lucky_participant')
  if (saved) {
    try {
      participant.value = JSON.parse(saved)
      registered.value = true
    } catch (e) {
      localStorage.removeItem('lucky_participant')
    }
  }

  fetchActivity().then(() => {
    // 获取到活动后，校验 localStorage 的参与者是否属于当前活动
    if (registered.value && participant.value.activityId !== activityId.value) {
      // 活动已变更，清除旧注册数据，重新显示注册表单
      localStorage.removeItem('lucky_participant')
      participant.value = {}
      registered.value = false
    }
  })
})
```

核心逻辑：
- 页面加载时先读 localStorage（快速展示已有数据）
- `fetchActivity()` 完成后，对比 `participant.activityId` 和当前 `activityId`
- 不匹配 → 清除旧数据，`registered = false`，重新显示注册表单

## 用户需要操作
1. **重启 lucky-mobile 前端**（`npm run dev`）
2. **手机端清除缓存**或直接重新扫码
3. 手机页面会检测到活动变更，自动显示注册表单
4. 重新填写注册信息 → 参与人数会正常更新

## 结果
- 换活动后，手机端会自动要求重新注册
- 注册后参与人数正常更新
- localStorage 中的数据始终属于当前活动
