# 配置 Claude Code Status Line

## 提示词
用户说 statusline 没有显示了，请检查当前的 statusline 配置状态，看看是否配置正确，如果没有配置就帮用户配置上。

## 做了什么事情

1. **检查当前配置状态**
   - 读取 `C:\Users\28182\.claude\settings.json` 文件
   - 发现配置文件中没有 `statusLine` 配置项

2. **检查用户的 shell 配置文件**
   - 尝试读取 `.bashrc` 和 `.zshrc` 文件
   - 两个文件都不存在，无法获取 PS1 设置

3. **创建 statusline 脚本文件**
   - 在 `C:\Users\28182\.claude\statusline-command.sh` 创建了 bash 脚本
   - 脚本功能：
     - 显示当前工作目录（最后两级路径）
     - 显示 Git 分支名（如果在 git 仓库中）
     - 显示当前使用的模型名称
     - 显示会话名称（如果设置了）
     - 显示上下文使用百分比

4. **更新 settings.json 配置**
   - 在配置文件中添加了 `statusLine` 配置项
   - 配置为命令类型，指向创建的脚本文件

## 为什么这样做

- **检查现有配置**：首先需要确认问题的根本原因，是因为配置缺失还是配置错误
- **创建脚本文件**：使用脚本文件而非直接内联命令，便于后续维护和修改
- **显示有用信息**：statusline 应该显示对用户最有帮助的信息，包括：
  - 工作目录：帮助用户了解当前所在位置
  - Git 分支：开发时经常需要知道当前分支
  - 模型信息：确认使用的 AI 模型
  - 上下文使用率：监控 token 使用情况，避免超出限制

## 产生的结果

### 配置文件变更

**`C:\Users\28182\.claude\settings.json`** 新增配置：
```json
"statusLine": {
  "type": "command",
  "command": "bash C:/Users/28182/.claude/statusline-command.sh"
}
```

**`C:\Users\28182\.claude\statusline-command.sh`** 新建脚本：
- 脚本会读取 Claude Code 通过 stdin 传入的 JSON 数据
- 解析并格式化显示：工作目录 | Git分支 | 模型名 | 上下文使用率

### Status Line 显示效果

示例输出：
```
java/lucky | main | mimo-v2.5-pro | ctx:25%
```

### 后续操作

1. **重启 Claude Code** 以使配置生效
2. 如果需要修改 statusline 显示内容，可以编辑 `C:\Users\28182\.claude\statusline-command.sh` 文件
3. 可用的 JSON 字段包括：
   - `.workspace.current_dir` - 当前工作目录
   - `.model.display_name` - 模型显示名
   - `.context_window.used_percentage` - 上下文使用百分比
   - `.session_name` - 会话名称
   - `.workspace.git_worktree` - Git worktree 名称

## 注意事项

- 脚本使用了 `jq` 命令解析 JSON，需要确保系统中已安装 jq
- Git 命令使用了 `--no-optional-locks` 参数避免潜在的锁冲突
- 路径使用正斜杠 `/` 而非反斜杠 `\`，兼容 bash 环境
