# 后端 — 新增屏幕配置数据层与文件上传API

## 用户问题（提示词）
请完成后端任务，严格遵循项目现有代码模式，包括：
1. 在 schema.sql 中新增 screen_config 表
2. 创建 ScreenConfig 实体类、Mapper、Service 接口和实现
3. 创建文件上传工具类 FileUploadUtil
4. 创建 ScreenConfigController API
5. 创建 WebMvcConfig 静态资源映射

## 如何分析的

### 1. 项目代码模式调研
- 读取了 `schema.sql`，了解现有表结构（activity、participant、lottery_round、winner、danmaku、sensitive_word）
- 读取了 `Activity.java` 实体类，确认模式：`@Data` + `@TableName` + `@TableId(type = IdType.AUTO)`
- 读取了 `ActivityMapper.java`，确认模式：极简风格，仅继承 `BaseMapper<Entity>`
- 读取了 `ActivityService.java` / `ActivityServiceImpl.java`，确认 Service 模式：接口继承 `IService<Entity>`，实现继承 `ServiceImpl<Mapper, Entity>`
- 读取了 `ActivityController.java`，确认 Controller 模式：`@RestController` + `@RequestMapping("/api/xxx")` + `@RequiredArgsConstructor` + `Result<T>` 统一返回
- 读取了 `CorsConfig.java` / `WebSocketConfig.java`，确认 Config 模式：`@Configuration`
- 读取了 `Result.java`，确认返回包装类使用 `Result.ok()` / `Result.error()`

### 2. 设计决策
- 完全遵循现有项目的代码风格和注解模式
- screen_config 表作为单例配置表（id=1），用于存储屏幕背景类型和URL
- FileUploadUtil 作为静态工具类处理文件上传/删除/类型校验
- WebMvcConfig 映射 `/uploads/**` 到本地 `uploads/` 目录实现静态资源访问

| 方法 | 路径 | 说明 |
|------|------|------|

| POST | `/api/screen/background` | 上传背景文件（type + file） |
| DELETE | `/api/screen/background` | 清除背景配置 |

### 文件上传特性
- 支持图片格式：jpg、png、gif、webp
- 支持视频格式：mp4、webm、ogg
- 文件存储路径：`uploads/` 目录（项目根目录）
- 文件名：UUID + 原扩展名，避免冲突
- 上传新文件时自动删除旧文件
- 清除背景时同时删除文件和数据库记录

## 为什么这样做

1. **遵循现有模式** — 新代码与项目现有 Entity/Mapper/Service/Controller 的写法完全一致，确保代码风格统一
2. **单例配置设计** — screen_config 固定 id=1，使用 `saveOrUpdate` 简化了配置管理
3. **文件校验前置** — Controller 层先校验文件类型再调用 Service，层次职责分明
4. **WebMvcConfigurer** — 通过 Spring MVC 配置映射静态资源，无需额外 Controller 处理文件访问
5. **@RequiredArgsConstructor** — 使用构造器注入，与现有 Controller 风格保持一致
