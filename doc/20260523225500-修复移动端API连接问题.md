# 20260523225500-修复移动端API连接问题

## 问题提示词

用户反馈移动端出现错误：
```
GET http://172.27.108.225:8080/api/activity/current net::ERR_CONNECTION_REFUSED
```

## 问题分析

这是一个网络连接错误，移动端应用尝试连接后端服务器时被拒绝。

### 原因定位

1. 检查移动端环境配置文件 `.env.development`，发现配置的 API 地址是 `http://172.27.108.225:8080`
2. 使用 `ipconfig` 命令检查当前机器的实际 IP 地址
3. 发现当前机器的 IP 地址是 `172.27.59.47`，与配置的 IP 地址不匹配

**根本原因**：IP 地址配置错误，导致移动端无法连接到后端服务器。

## 解决方案

### 1. 更新移动端环境配置

修改 `lucky-mobile/.env.development` 文件：
```diff
- VITE_API_BASE=http://172.27.108.225:8080
- VITE_WS_URL=ws://172.27.108.225:8080/ws/lucky
+ VITE_API_BASE=http://172.27.59.47:8080
+ VITE_WS_URL=ws://172.27.59.47:8080/ws/lucky
```

### 2. 更新后端配置

修改 `lucky-server/src/main/resources/application.yml` 文件：
```diff
  lucky:
-   base-url: http://172.27.108.225:5174
+   base-url: http://172.27.59.47:5176
```

### 3. 重启服务

1. 停止当前运行的后端 Java 进程
2. 重启移动端开发服务器（端口变为 5176）
3. 重启后端服务器

## 修复结果

1. ✅ 移动端 API 地址配置正确
2. ✅ 后端服务器成功启动在 8080 端口
3. ✅ 移动端开发服务器成功启动在 5176 端口
4. ✅ WebSocket 连接正常建立

## 修改文件

- `lucky-mobile/.env.development` - 更新 API 和 WebSocket 地址
- `lucky-server/src/main/resources/application.yml` - 更新 base-url 配置

## 注意事项

IP 地址可能会因网络环境变化而改变（如重启电脑、切换网络等）。如果再次遇到连接问题，请使用 `ipconfig` 命令检查当前机器的 IP 地址，并更新配置文件。
