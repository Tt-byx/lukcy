# 20260523120000-修复移动端刷新跳转逻辑

## 问题提示词

用户反馈：该项目的mobile端有bug，具体为在没创建活动的情况下，点击刷新会跳转到输入学号的页面，但一会会跳转到暂无活动的页面，在有活动的情况下，点击刷新同样会跳转到输入学号的界面，然后停留时间长一点后又跳转到暂无活动。需要的是没有活动的话就是显示暂无活动，有活动的话就显示登陆参与活动或者学号快速进入。

## 分析过程

### 1. 问题定位

查看 `lucky-mobile/src/App.vue` 文件，发现模板中有三个条件判断：
- `v-if="registered"` - 显示直播间界面
- `v-else-if="noActivity"` - 显示暂无活动
- `v-else` - 显示注册表单（学号输入页面）

### 2. 原代码逻辑问题

原代码在 `onMounted` 中的逻辑：
```javascript
onMounted(() => {
  // 先检查 localStorage，再根据活动校验
  const saved = localStorage.getItem('lucky_participant')
  if (saved) {
    try {
      participant.value = JSON.parse(saved)
      registered.value = true  // 立即设置为 true
    } catch (e) {
      localStorage.removeItem('lucky_participant')
    }
  }

  fetchActivity().then(() => {
    // 校验 localStorage 的参与者是否属于当前活动
    if (registered.value && participant.value.activityId !== activityId.value) {
      localStorage.removeItem('lucky_participant')
      participant.value = {}
      registered.value = false
    } else if (registered.value) {
      connectWebSocket()
      fetchScreenConfig()
    }
  })
})
```

**问题所在：**
1. 首先从 localStorage 读取数据，如果有就立即设置 `registered = true`
2. 这会导致模板先显示"直播间界面"
3. 然后异步调用 `fetchActivity()` 获取活动状态
4. 如果没有活动，`noActivity` 被设置为 `true`，但 `registered` 还是 `true`
5. 由于模板中 `v-if="registered"` 优先级高于 `v-else-if="noActivity"`，所以还是显示直播间
6. 之后某些逻辑（可能是活动变更检测）将 `registered` 设置为 `false`，才显示"暂无活动"

这就解释了为什么用户会看到页面先跳转到输入学号界面，然后才跳转到暂无活动页面。

## 解决方案

修改 `onMounted` 中的逻辑，确保：
1. **先获取活动状态**，再决定是否恢复 localStorage 中的注册数据
2. **如果没有活动**，立即清除 localStorage 和 registered 状态，直接显示"暂无活动"
3. **如果有活动**，检查 localStorage 中的参与者是否属于当前活动，只有活动ID匹配时才恢复注册状态

### 修改后的代码

```javascript
onMounted(() => {
  fetchActivity().then(() => {
    if (noActivity.value) {
      localStorage.removeItem('lucky_participant')
      participant.value = {}
      registered.value = false
      return
    }

    const saved = localStorage.getItem('lucky_participant')
    if (saved) {
      try {
        const savedParticipant = JSON.parse(saved)
        if (savedParticipant.activityId === activityId.value) {
          participant.value = savedParticipant
          registered.value = true
          connectWebSocket()
          fetchScreenConfig()
        } else {
          localStorage.removeItem('lucky_participant')
          participant.value = {}
          registered.value = false
        }
      } catch (e) {
        localStorage.removeItem('lucky_participant')
        participant.value = {}
        registered.value = false
      }
    }
  })
})
```

## 修改结果

1. ✅ 修复了刷新时页面闪烁的问题
2. ✅ 没有活动时直接显示"暂无活动"页面
3. ✅ 有活动时显示登录/学号快速进入页面
4. ✅ 活动变更时自动清除旧的注册数据
5. ✅ 项目构建成功，无编译错误
