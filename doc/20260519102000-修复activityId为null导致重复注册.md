# 修复 activityId 为 null 导致重复注册问题

## 用户问题
用户展示了后端日志：
```
SELECT COUNT(*) FROM participant WHERE (activity_id = ? AND student_id = ?)
Parameters: null, 20241004574     ← activity_id 是 null！
INSERT INTO participant ( name, phone, student_id, status ) VALUES ...  ← 没有 activity_id 列！
（同一学号 20241004574 被重复 INSERT 了两次）
```

## 分析过程
1. 日志显示 `activity_id` 参数为 `null`，SELECT 查重永远返回 0
2. INSERT 语句中也没有 `activity_id` 列（MyBatis-Plus 默认跳过 null 字段）
3. 同一学号被重复插入两次，证明查重完全失效
4. 追踪代码链路：
   - 前端 `fetchActivity()` → 调用 `/api/activity/current`
   - 后端 `getCurrentActivity()` → 查询 status=0 或 1 的最新活动
   - 如果数据库没有活动，返回 `{"code": 500, "message": "暂无活动"}`
   - 前端 `data.code !== 200`，`activityId` 保持 `null`
   - 手机端注册表单仍然显示，用户点击注册，`activityId` 为 null 直接发到后端

**根因：用户没有通过管理后台创建活动，手机端直接注册导致 activityId 为 null。**

## 修改内容

### 1. lucky-mobile/src/App.vue（前端防护）
- 新增 `noActivity` 状态变量
- `fetchActivity()` 失败时设置 `noActivity = true`
- 无活动时显示"暂无活动"提示页，不展示注册表单
- 注册按钮增加 `:disabled="registering || !activityId"` 禁用条件

### 2. ParticipantServiceImpl.java（后端防护）
- `register()` 方法开头增加 `activityId` 空值校验
- 为 null 时抛出 `"活动不存在，请先创建活动"` 异常

### 3. DanmakuServiceImpl.java（后端防护）
- `sendDanmaku()` 方法开头增加 `activityId` 空值校验

### 4. LotteryServiceImpl.java（后端防护）
- `createRound()` 方法开头增加 `activityId` 空值校验

## 用户接下来需要做的事

### 第一步：清理脏数据
之前 activity_id 为 null 的重复注册数据需要清理：
```sql
DELETE FROM participant WHERE activity_id IS NULL;
```

### 第二步：重启后端
在 lucky-server 目录重新执行 `mvn spring-boot:run`

### 第三步：用管理后台创建活动
1. 访问 `http://localhost:5175`（管理后台）
2. 在"活动管理"页输入活动名称，点击创建
3. 点击"开始活动"

### 第四步：手机端正常注册
访问 `http://172.27.31.152:5174`，此时应该能看到活动名称和注册表单，注册时 activityId 会正确传入。

## 修复效果
- 未创建活动时：手机端显示"暂无活动"，无法注册
- 创建活动后：正常注册，查重正常工作
- 即使前端绕过（直接调API）：后端也会校验 activityId 并返回错误
