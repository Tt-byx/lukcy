# 20260520001500-手机端直播间改造与在线人数修复

## 问题提示词

用户反馈：
1. 手机扫码后参与人数增加了，但在线人数没增加
2. 希望手机端登录后的界面像直播间，上面有消息滚动显示谁发了弹幕，下面有输入框可以发弹幕
3. 手机端背景可以自己设置
4. 在直播界面的话就算一个在线人数
5. 点击开始抽奖时，screen屏幕的动画和阴影没有铺满

## 做了什么事情

### 1. 在线人数修复

**问题诊断**：手机端没有 WebSocket 连接，只用 HTTP API。而 `LuckyWebSocketHandler` 的在线人数只统计 WebSocket 连接数。

**修改文件**：
- `lucky-server/.../websocket/LuckyWebSocketHandler.java`
  - 在 `afterConnectionEstablished` 和 `afterConnectionClosed` 中添加 `broadcastOnlineCount()` 调用
  - 新增 `broadcastOnlineCount()` 方法，广播 `online_update` 消息给所有客户端
- `lucky-screen/src/App.vue`
  - WebSocket 消息处理添加 `online_update` case，实时更新 `onlineCount`

### 2. 弹幕广播携带发送者名字

**问题诊断**：原来广播弹幕时只发送 `Danmaku` 实体（只有 `participantId`），没有发送者名字。

**修改文件**：
- `lucky-server/.../service/impl/DanmakuServiceImpl.java`
  - 注入 `ParticipantMapper`
  - 新增 `broadcastDanmaku()` 方法，构建包含 `participantName` 的 Map 进行广播
  - `sendDanmaku()` 和 `reviewDanmaku()` 都改用 `broadcastDanmaku()`

### 3. 手机端背景配置

**新增功能**：允许管理后台单独为手机端设置背景。

**修改文件**：
- `lucky-server/.../entity/ScreenConfig.java` - 新增 `mobileBackgroundType`、`mobileBackgroundUrl` 字段
- `lucky-server/.../config/DatabaseInitializer.java` - ALTER TABLE 添加手机端背景列
- `lucky-server/.../service/ScreenConfigService.java` - 新增接口方法
- `lucky-server/.../service/impl/ScreenConfigServiceImpl.java` - 实现手机端背景上传/清除
- `lucky-server/.../controller/ScreenConfigController.java` - 新增 `POST/DELETE /api/screen/mobile-background` 端点
- `lucky-admin/src/App.vue` - 屏幕设置 tab 添加手机端背景上传 UI

### 4. 手机端直播间 UI 重写

**整体改造**：将登录后的界面从简单的弹幕发送页改为直播间风格。

**修改文件**：
- `lucky-mobile/src/App.vue` - 完整重写
  - 新增 WebSocket 连接（进入直播间时建立，离开时断开）
  - 顶部栏：显示活动名称 + 在线人数
  - 消息滚动区：实时显示弹幕（发送者: 内容），自动滚动到底部
  - 底部输入栏：快捷弹幕 + 输入框
  - 支持手机端背景配置（图片/视频）
- `lucky-mobile/.env.development` - 添加 `VITE_WS_URL`
- `lucky-mobile/.env.production` - 添加 `VITE_WS_URL`

### 5. 抽奖动画全屏修复

**问题诊断**：`.lottery-rolling` 和 `.lottery-result` 使用 `position: absolute`，只覆盖 `.right-panel` 而非整个屏幕。

**修改文件**：
- `lucky-screen/src/App.vue`
  - `.lottery-rolling` 改为 `position: fixed; z-index: 150`
  - `.lottery-result` 改为 `position: fixed; z-index: 200`

## 为什么这样做

1. **WebSocket 统计在线人数**：WebSocket 连接是实时双向的，连接建立/断开都能立即感知，比 HTTP 轮询更准确
2. **弹幕携带发送者名字**：手机端直播间需要显示"谁发了什么"，需要在广播时携带 `participantName`
3. **手机端背景独立配置**：大屏和手机端的显示比例不同，独立配置可以分别优化视觉效果
4. **position: fixed 全屏覆盖**：`fixed` 定位相对于视口，不受父元素定位上下文限制，确保遮罩覆盖整个屏幕

## 产生的结果

- 手机进入直播间后自动建立 WebSocket 连接，计入在线人数
- 大屏在线人数实时更新（连接/断开时自动广播）
- 手机端直播间风格：顶部活动名+在线人数，中间消息流，底部输入栏
- 弹幕显示发送者名字（如 "张三: 666"）
- 管理后台可单独为手机端设置背景
- 抽奖动画遮罩覆盖整个大屏（包括左侧二维码区域）

## 验证方案

1. 手机打开直播间 → 大屏 `online` 数字实时增加
2. 手机关闭页面 → 大屏 `online` 数字减少
3. 手机发送弹幕 → 手机端消息区显示"名字: 内容"
4. 管理后台上传手机端背景 → 手机端直播间背景更新
5. 管理后台点击开始抽奖 → 遮罩覆盖整个大屏
