# 修复二维码 localhost 问题 - 改用 Vite 环境变量配置

## 用户问题
用户将 `application.yml` 中的 `base-url` 改为了 `http://172.27.31.152:5174`，但微信扫码后跳转的仍然是 localhost。用户选择了改用环境变量配置的方式，并担心本地测试和线上测试如何切换。

## 分析过程
1. 搜索了整个项目的 `localhost` 硬编码位置
2. 发现二维码不是在**后端**生成的，而是在 `lucky-screen`（大屏前端）中用 `qrcode` 库直接从前端生成的
3. 三个前端全部存在 `localhost` 硬编码问题：
   - **lucky-screen**：3处（API_BASE、WS_URL、MOBILE_URL）
   - **lucky-mobile**：1处（API_BASE）
   - **lucky-admin**：1处（API_BASE）
4. 后端 `application.yml` 中的 `lucky.base-url` 配置实际上**没有被前端读取**，改了也没用

## 修改内容

### 1. 修改三个 App.vue（共 3 个文件）
将硬编码的 `localhost` 改为读取 Vite 环境变量，并保留 localhost 作为兜底默认值：

**lucky-screen/src/App.vue**：
```javascript
// 之前
const API_BASE = 'http://localhost:8080'
const WS_URL = 'ws://localhost:8080/ws/lucky'
const MOBILE_URL = 'http://localhost:5174'

// 之后
const API_BASE = import.meta.env.VITE_API_BASE || 'http://localhost:8080'
const WS_URL = import.meta.env.VITE_WS_URL || 'ws://localhost:8080/ws/lucky'
const MOBILE_URL = import.meta.env.VITE_MOBILE_URL || 'http://localhost:5174'
```

**lucky-mobile/src/App.vue** 和 **lucky-admin/src/App.vue**：
```javascript
// 之前
const API_BASE = 'http://localhost:8080'

// 之后
const API_BASE = import.meta.env.VITE_API_BASE || 'http://localhost:8080'
```

### 2. 新建环境变量文件（共 6 个文件）

每个前端目录下创建了两个文件：

| 文件 | 说明 |
|------|------|
| `.env.development` | `npm run dev` 时自动加载，配置为电脑IP `172.27.31.152` |
| `.env.production` | `npm run build` 时自动加载，需改为实际服务器地址 |

**lucky-screen/.env.development**：
```
VITE_API_BASE=http://172.27.31.152:8080
VITE_WS_URL=ws://172.27.31.152:8080/ws/lucky
VITE_MOBILE_URL=http://172.27.31.152:5174
```

**lucky-mobile/.env.development** 和 **lucky-admin/.env.development**：
```
VITE_API_BASE=http://172.27.31.152:8080
```

## 为什么这样设计
- Vite 的 `.env` 机制天然支持按环境切换，`npm run dev` 自动加载 `.env.development`，`npm run build` 自动加载 `.env.production`
- 使用 `import.meta.env.VITE_xxx || '默认值'` 模式，即使没有 `.env` 文件也不会报错，兜底使用 localhost
- 每个前端独立的 `.env` 文件，互不影响

## 如何操作

### 本地测试（电脑 + 手机同 WiFi）
1. 确认 `172.27.31.152` 是你当前电脑的 IP（如果变了，修改 `.env.development`）
2. `npm run dev` 启动前端（会自动读取 .env.development）
3. 手机浏览器访问 `http://172.27.31.152:5174`
4. 大屏二维码现在会正确指向 `http://172.27.31.152:5174`

### 部署到线上
1. 把三个 `.env.production` 中的 `你的服务器IP` 改为实际服务器地址或域名
2. `npm run build` 打包（会自动读取 .env.production）
3. 部署打包后的文件到服务器

### 如果你的电脑 IP 变了
只需修改各 `.env.development` 文件中的 IP 地址，然后**重启**前端 dev server 即可生效（Vite 环境变量在 dev server 启动时读取，修改后需要重启）。
