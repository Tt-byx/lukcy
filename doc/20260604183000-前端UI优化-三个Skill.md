## 为什么这样做

### 1. **遵循设计原则**

#### frontend-design 原则
- 避免通用的"AI风格"设计
- 强调大胆的美学方向
- 注重排版、配色、动效、空间构图

#### impeccable 原则
- 颜色对比度检查（≥4.5:1）
- 排版层次优化（≥1.25比例）
- 动效要有意义，不是装饰
- 响应式设计必须支持

#### taste-skill 原则
- 先推断设计方向，再动手
- 使用三档配置系统
- 避免AI默认风格（紫色渐变、居中hero等）
- 每个设计都要有明确的美学意图

### 2. **提升用户体验**

**大屏端**：
- 沉浸式体验，吸引观众注意力
- 丰富的动效，营造喜庆氛围
- 清晰的信息层次，易于远距离观看

**手机端**：
- 简洁的界面，快速完成操作
- 友好的交互，降低使用门槛
- 舒适的视觉，长时间使用不疲劳

**管理后台**：
- 专业的界面，提升信任感
- 高效的布局，快速找到功能
- 清晰的反馈，减少操作错误

### 3. **技术实现**

**CSS变量系统**：
- 统一的设计token
- 易于维护和修改
- 支持主题切换

**现代CSS特性**：
- backdrop-filter 玻璃拟态
- CSS Grid/Flexbox 布局
- CSS动画/过渡
- 媒体查询响应式

**性能优化**：
- 使用CSS动画而非JS
- 合理的动画时长
- 避免重绘重排
