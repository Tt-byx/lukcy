# 修改 Trae CN 编辑器 JDK 配置为 Temurin 17

## 提示词

> 把 `c:\Users\28182\AppData\Roaming\Trae CN\User\settings.json` 里面的 java 和 maven 用的 jdk 都改成 17，地址在 `C:\Users\28182\.jdks\temurin-17.0.18\bin`

## 做了什么

修改了 Trae CN 编辑器的 `settings.json` 文件，将所有 Java 和 Maven 相关的 JDK 路径从 `jdk-25` 改为 `temurin-17.0.18`。

### 具体修改内容

| 配置项 | 修改前 | 修改后 |
|--------|--------|--------|
| `terminal.integrated.env.windows.JAVA_HOME` | `C:\Users\28182\.jdks\jdk-25` | `C:\Users\28182\.jdks\temurin-17.0.18` |
| `terminal.integrated.env.windows.PATH` | `C:\Users\28182\.jdks\jdk-25\bin` | `C:\Users\28182\.jdks\temurin-17.0.18\bin` |
| `java.configuration.runtimes` JavaSE-17 path | pleiades 扩展包的 JDK 17 路径 | `C:\Users\28182\.jdks\temurin-17.0.18` |
| `java.configuration.runtimes` default | JavaSE-26 | JavaSE-17 |
| `maven.terminal.customEnv.JAVA_HOME` | `C:\Users\28182\.jdks\jdk-25` | `C:\Users\28182\.jdks\temurin-17.0.18` |
| `java.import.gradle.java.home` | `C:\Users\28182\.jdks\jdk-25` | `C:\Users\28182\.jdks\temurin-17.0.18` |

## 为什么这样做

用户需要将开发环境统一切换到 JDK 17（Eclipse Temurin 发行版），以确保项目编译和运行使用一致的 JDK 版本。

## 产生的结果

1. **终端环境**：新打开的终端将自动使用 JDK 17，`JAVA_HOME` 指向 `temurin-17.0.18`
2. **Java 语言服务器**：默认运行时切换为 JavaSE-17
3. **Maven 构建**：Maven 命令将使用 JDK 17 执行
4. **Gradle 构建**：Gradle 导入将使用 JDK 17

**注意**：需要重启 Trae CN 编辑器才能使配置完全生效。
